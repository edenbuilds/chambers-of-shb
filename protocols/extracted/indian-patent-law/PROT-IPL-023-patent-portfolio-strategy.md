## Core Process

### Step 1: Define Portfolio Objectives

#### Strategic Goals:

| Objective | Portfolio Approach |
|-----------|-------------------|
| **Market exclusion** | Broad claims, key markets |
| **Licensing revenue** | Volume, diverse applications |
| **Defensive** | Design-arounds, blocking patents |
| **Cross-licensing** | Complementary technologies |
| **Investor value** | Visible, validated assets |
| **Competitive signaling** | Strategic publication |

#### India-Specific Considerations:
- Large market, cost-effective filing
- Section 3 limitations (pharma, software)
- Compulsory licensing risk (essential medicines)
- Growing enforcement environment

### Step 2: Portfolio Building Strategies

#### Filing Types:

| Type | When to Use |
|------|-------------|
| **Ordinary (Section 7)** | India-first, no foreign filing planned |
| **Convention (Section 135)** | Following foreign priority filing |
| **PCT National Phase (Section 7(1A))** | International strategy |
| **Divisional (Section 16)** | Separate distinct inventions |
| **Patent of Addition (Section 54)** | Improvements to parent patent |

#### Technology Coverage:

```
Core Technology
├── Foundational Patents (broad claims)
├── Improvement Patents (enhancements)
├── Application Patents (specific uses)
├── Process Patents (manufacturing)
└── Design-around Blockers (adjacent space)
```

#### Claim Strategy:
- **Independent claims**: Maximum scope
- **Dependent claims**: Fallback positions
- **Product claims**: Composition/device
- **Method claims**: Process coverage
- **Use claims**: Where available (not India for known drugs)

### Step 3: Portfolio Analysis Framework

#### Quantitative Metrics:

| Metric | What It Measures |
|--------|------------------|
| **Total patents** | Portfolio size |
| **Grant rate** | Filing quality |
| **Citation count** | Technical impact |
| **Claim count** | Scope breadth |
| **Remaining life** | Portfolio maturity |
| **Cost per patent** | Efficiency |
| **Revenue per patent** | Monetization |

#### Qualitative Assessment:

| Factor | Assessment |
|--------|------------|
| **Technology coverage** | Core vs. peripheral |
| **Competitive overlap** | Freedom to operate |
| **Enforcement strength** | Litigation viability |
| **Market alignment** | Business relevance |
| **Validity strength** | Invalidity risk |

### Step 4: Competitive Intelligence

#### Competitor Analysis:

1. **Patent monitoring**:
   - Set up alerts on InPASS
   - Track competitor filings
   - Monitor publication dates

2. **Landscape mapping**:
   - Technology areas covered
   - White space identification
   - Blocking opportunities

3. **Strength assessment**:
   - Portfolio size comparison
   - Claim scope analysis
   - Enforcement history

#### Tools for Indian Patents:
- InPASS (Indian Patent Advanced Search)
- Global Patent Index (Espacenet)
- Commercial databases (Orbit, PatSnap)

### Step 5: Filing Decision Framework

#### File Patent When:

| Criteria | Threshold |
|----------|-----------|
| **Novelty** | Prior art clear |
| **Commercial value** | Revenue potential |
| **Enforceability** | Can detect infringement |
| **Section 3 clearance** | Not excluded |
| **Cost-benefit** | ROI positive |

#### Consider Trade Secret When:
- Difficult to reverse engineer
- Long useful life expected
- Detection of infringement difficult
- Patent protection weak (Section 3)

#### Don't File When:
- Likely Section 3 rejection
- No commercial application
- Detection impossible
- Prior art exists
- Disclosure harmful

### Step 6: Portfolio Maintenance

#### Annual Review Process:

1. **Renewal decisions**:
   - Assess continued relevance
   - Calculate ROI per patent
   - Compare to technology roadmap

2. **Abandonment criteria**:
   - No commercial use/licensing
   - Technology obsolete
   - High maintenance vs. value
   - Superseded by improvement

3. **Enhancement opportunities**:
   - File divisionals
   - File patents of addition
   - Expand geographic coverage

#### Cost Management:

| Strategy | Savings |
|----------|---------|
| **Entity status** | 50-80% on fees |
| **Timely renewals** | Avoid surcharges |
| **Strategic abandonment** | Eliminate dead weight |
| **E-filing** | Lower official fees |
| **Batch processing** | Agent efficiencies |

### Step 7: Portfolio Optimization

#### Pruning Strategy:
```
For each patent, assess:
├─ Current business relevance → Low = Candidate
├─ Licensing potential → Low = Candidate
├─ Defensive value → Low = Candidate
├─ Remaining term → Short = Consider
└─ Maintenance cost → High vs. value = Candidate

If multiple "Candidate" flags → Abandon or Sell
```

#### Gap Filling:
- Identify uncovered technology areas
- File new applications
- Acquire third-party patents
- License-in complementary IP

#### Monetization:
- Licensing programs
- Patent sales
- Enforcement actions
- Cross-licensing deals

### Step 8: Budget Allocation

#### Typical Cost Components:

| Phase | Cost Range (INR) |
|-------|------------------|
| **Prior art search** | ₹15,000 - 50,000 |
| **Drafting** | ₹50,000 - 2,00,000 |
| **Filing fees** | ₹1,600 - 8,000 |
| **Prosecution** | ₹30,000 - 1,00,000 |
| **Grant fee** | ₹4,000 - 20,000 |
| **Annual renewal** | ₹800 - 80,000 |

*Ranges reflect natural person/startup to large entity*

#### Budget Allocation Model:
```
Total IP Budget: 100%
├── New filings: 50-60%
├── Prosecution: 15-20%
├── Maintenance: 15-20%
├── FTO/searches: 5-10%
└── Enforcement reserve: 5-10%
```

## Indian Patent Portfolio Best Practices

### Do's:
- File early (first-to-file system)
- Claim startup/small entity benefits
- Use expedited examination when eligible
- Monitor competitors via InPASS alerts
- Maintain Form 27 compliance
- Consider divisionals for broad inventions

### Don'ts:
- Over-rely on software patents (Section 3(k) risk)
- Ignore Section 3(d) for pharma (Novartis test)
- Miss renewal deadlines (restoration expensive)
- File without prior art search
- Neglect working statement (Form 27)

## Common Portfolio Mistakes

1. **Filing without strategy**: Align with business
2. **Ignoring entity benefits**: Claim proper status
3. **Passive maintenance**: Regular review needed
4. **No competitive monitoring**: Miss blocking opportunities
5. **Over-filing**: Quality over quantity
6. **Under-filing**: Miss protection opportunities
